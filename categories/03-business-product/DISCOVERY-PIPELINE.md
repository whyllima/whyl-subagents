# Discovery Pipeline: Product → UX → Business → Content

Fluxo de descoberta e validação entre os quatro agentes. Cada um tem um papel claro na cadeia **ideia → problema real → requisitos/viabilidade → posicionamento**.

## Fluxo

```
product-manager          ux-researcher           business-analyst        content-marketer
(O quê e por quê)   →   (Problema é real?)   →   (Requisitos + viabilidade)   →   (Posicionamento e narrativa)
     │                          │                          │                              │
     │ Gera/valida ideia        │ Confirma dor do usuário  │ Traduz em requisitos         │ Narrativa e mensagem
     │ Hipótese de valor        │ Severidade, frequência   │ Viabilidade técnica/negócio  │ Posicionamento de mercado
     ▼                          ▼                          ▼                              ▼
  Problem statement      Problem validation         Requirements doc           Positioning + messaging
  Value hypothesis       Pain assessment           Feasibility report          Narrative arc
```

## Papéis e handoffs

| Agente | Papel | Entrada | Saída para o próximo |
|--------|--------|---------|----------------------|
| **product-manager** | Gera e valida a ideia (o quê e por quê) | Contexto de mercado, visão | Problem statement, value hypothesis, alvos de validação |
| **ux-researcher** | Confirma se o problema é real (dor do usuário) | Hipótese e problem statement do PM | Problem validation report, pain severity, user quotes |
| **business-analyst** | Traduz a ideia em requisitos e viabilidade | Ideia validada + problema confirmado | Requisitos, viabilidade, escopo, critérios de sucesso |
| **content-marketer** | Posicionamento e narrativa | Value prop, requisitos, diferenciação | Positioning statement, narrative, messaging hierarchy |

## Melhorias aplicadas nos agentes

- **product-manager**: Foco explícito em idea generation + validation; artefatos de “o quê e por quê”; handoff para ux-researcher.
- **ux-researcher**: Foco em “problema é real?” e dor do usuário; pain assessment; handoff para business-analyst.
- **business-analyst**: Entrada = ideia + problema validados; saída = requisitos + viabilidade; handoff para content-marketer.
- **content-marketer**: Entrada = value prop e requisitos; foco em posicionamento e narrativa (não só SEO/calendário).
